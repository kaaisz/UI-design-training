Understanding Selector

Child Selector

Matches an element that is a direct child of another

div > p { } Targets any < a > elements that are children of an < li > element (but not other < a > elements in the page)

See the Pen <for understanding&rt; Child selector by suzydp (@suzydp) on CodePen.

Descendant Selector

Matches an element that is a descendent of another specified element (not just a direct child of that element)

div p {} Targets any < a > elements that sit inside a < p > element, even if there are other elements nested between them

See the Pen zJaexo by suzydp (@suzydp) on CodePen.

General Sibling Selector

Matches an element that is a sibling of another, although it does not have to be the directly preceding element

div~p {} If you had two < p > elements that are siblings of an < h1 > element, this rule would apply to both
